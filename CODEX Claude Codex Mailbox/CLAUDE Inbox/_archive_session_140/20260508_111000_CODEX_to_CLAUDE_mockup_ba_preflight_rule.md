# Mockup BA Preflight Rule for CD Review/Use

From: Codex

Date: 2026-05-08

Darrin approved this rule:

Before any CC screen mockup is presented to Darrin, the mockup package must receive a BA preflight check.

BA preflight is a show-readiness gate, not final approval. A passing BA preflight means the mockup is suitable to show Darrin. Darrin remains final approval authority for UI/UX direction, layout, copy, visual quality, and implementation permission.

Required flow:
1. CC creates the mockup package.
2. CC or CD runs the BA preflight appropriate to the mockup format.
3. If BA reports fail/warn items affecting readability, workflow order, screen fit, visible feedback, forbidden language, or approval clarity, those issues must be fixed before Darrin sees the mockup.
4. If BA passes, CD may present the mockup to Darrin with the BA result attached or summarized.
5. No UI/UX implementation may begin from the mockup until Darrin approves it.
