## Bug #234 — Open Latest Report: ERR_CONNECTION_REFUSED

**Root cause:** The button navigates to `127.0.0.1:8791/api/ba_qa/latest`
— a local server endpoint that is not running.

**Fix:** Replace the HTTP navigation with a direct file open:

```python
from PySide6.QtGui import QDesktopServices
from PySide6.QtCore import QUrl

# In the button handler:
QDesktopServices.openUrl(QUrl.fromLocalFile(report_path))
```

Where `report_path` is the path to the latest report HTML/JSON file
on disk. Identify the canonical report output path from the existing
report-writer code (look near where the last-run timestamp is stored
— the same location should have the report file path).

If the server endpoint intentionally serves dynamic content (i.e., it
is not just a static file), flag this in your Step 0 RTC and propose
an alternative before touching the button handler.

**AC:**
- [ ] Clicking "Open Latest Report" opens the latest report file
      in the system default browser or viewer
- [ ] No network request to 127.0.0.1:8791
- [ ] If no report exists yet, button is disabled or shows a clear
      "No report available" message (not a browser error)

## Bug #235 — Layout: Open Latest Report isolated; RUN COMMAND console cramped

**Root cause:** "Open Latest Report" sits in its own horizontal band
separate from "Run QA" / "Run Screenshots," wasting a full row of
vertical space. The RUN COMMAND log console at the bottom is squeezed.

**Fix:** Move "Open Latest Report" onto the same row as "Run QA" and
"Run Screenshots" (right-aligned within the row). Remove the now-empty
horizontal band. Allow the RUN COMMAND panel to expand into the freed
vertical space — either via splitter or by removing a fixed height
constraint on the console area.

The last-run timestamp may remain above the button row or inline with
it — your call based on what fits cleanly.

**AC:**
- [ ] "Run QA", "Run Screenshots", and "Open Latest Report" all on
      one row
- [ ] No empty horizontal band where "Open Latest Report" previously
      sat
- [ ] RUN COMMAND log console is visibly taller than before
- [ ] Layout correct at typical window widths (1366px and 1750px)
