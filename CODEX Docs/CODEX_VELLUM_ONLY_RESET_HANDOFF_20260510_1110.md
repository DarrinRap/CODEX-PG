## HTML Mockup Accuracy Guidance

Darrin asked whether it is impossible to create an accurate screen mockup using HTML. Answer given:

- It is not impossible; HTML can be very accurate for layout, visual hierarchy, spacing, typography, controls, states, and responsive variants.
- HTML is weaker as an exact prediction of PySide6/QWidget behavior: native controls, font smoothing, scrollbars, splitters, dock widgets, menus, canvas/image behavior, and state complexity differ.
- For PG/Vellum, HTML mockups should be treated as a visual contract and measurement source, not proof that the PySide implementation will match.
- Best workflow: HTML mockup -> implementation -> screenshot comparison -> fix deltas -> repeat.
- Do not declare victory at the mockup stage.
