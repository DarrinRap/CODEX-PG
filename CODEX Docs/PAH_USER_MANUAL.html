PANDA Agent Hub

User Manual

PAH is the local cockpit for coordinating Codex, Claude Desktop, Claude Code, and Darrin through shared mailbox files under C:\CODEX PG. This guide explains what it does, how to use each feature, and where the safety boundaries are.

PAH is local-first and file-based. It does not call OpenAI, Anthropic, Claude Code headless, SMS providers, paid APIs, or Panda Gallery runtime code by itself.

Quick Start

Start PAH

From PowerShell, run the regular launcher:

& "C:\CODEX PG\CODEX Agent Hub\CODEX_start_agent_hub.ps1"

Or use the tray launcher when you want dashboard and log shortcuts:

& "C:\CODEX PG\CODEX Agent Hub\CODEX_start_agent_hub_tray.ps1"

Open Dashboard

Default browser URL:

http://127.0.0.1:8765

If port 8765 is busy, PAH chooses another local port and prints it in the terminal.

Mental Model

Think of PAH as a control room over a set of folders. Agents communicate by writing Markdown files into inboxes. PAH reads those files, validates them, groups them into threads, and exposes safe actions in one dashboard.

Graphic snippet: PAH sits over mailbox files. It is a cockpit, not a live agent executor.

Safety Gate

Daily Workflow

Morning or Session Start

  1. Launch PAH.
  2. Open the dashboard URL.
  3. Check unread messages, Darrin Queue, and actionable validation issues.
  4. Run Diagnostics if you are unsure the file bridge is healthy.
  5. Use Latest or Threads to decide what needs attention first.

When You Need Claude or Claude Code

  1. Use Compose on the right side.
  2. Choose Codex to Claude or Codex to Claude Code.
  3. Set a short subject, optional thread ID, optional reply-to reference, and the message body.
  4. Send the message. PAH writes a Markdown file into the target inbox.
  5. Refresh later and look for a reply in Latest or the relevant thread.

When Something Looks Wrong

  1. Open the Validator tab.
  2. Look at actionable issues first.
  3. Use quarantine only for mailbox messages that are malformed, spoofed, unsafe, or flooded.
  4. Open Safety if the issue involves approvals, adapters, quarantine, or protected action boundaries.

Messages and Threads

PAH reads the active mailbox folders and shows messages in a normalized shape. New PAH-composed messages use schema v1 frontmatter. Older legacy messages are still read where possible.

Latest

Best for scanning the newest traffic. Each row can show badges such as unread, decision, important, or validation-related status.

Threads

Best for following a conversation. PAH groups by Thread-ID first, then falls back to reply and message IDs.

Message Lifecycle

Darrin Queue

The Darrin Queue is PAH's high-signal attention list. It is generated from message metadata and saved at:

C:\CODEX PG\CODEX Claude Codex Mailbox\CODEX_DARRIN_DECISIONS_NEEDED.md

What Adds an Item

  • requires_darrin_decision: true
  • thread_status: waiting_on_darrin
  • action_owner: darrin
  • priority: urgent
  • Any approval_boundary containing _requires_darrin

How to Use It

  1. Open Darrin Queue.
  2. Read active items first.
  3. Resolve, supersede, or dismiss stale local queue items when appropriate.
  4. Remember: changing queue state does not edit the original mailbox message.

Validator and Quarantine

The Validator tab separates actionable problems from old or accepted legacy noise. It helps you catch mailbox poisoning, source spoofing, duplicate IDs, schema drift, and unsafe boundaries.

Actionable Validation

Items that can affect current coordination. Look here first.

Inactive Validation

Accepted legacy, resolved, or dismissed findings. These remain visible for context.

Quarantine Rules

PAH never auto-quarantines during refresh. Quarantine is explicit and requires confirmation. Use it for files that should not remain active in the mailbox.

Work Board

The Work Board is local coordination state for tasks assigned to Codex, Claude Desktop, or Claude Code. It does not launch agents by itself.

Create a Work Item

  1. Use the Work Item card on the right rail.
  2. Choose owner, priority, title, and summary.
  3. Click Create Work Item.

Dispatch a Work Item

  1. Open the Work tab.
  2. Dispatch the item to Claude Desktop or Claude Code.
  3. PAH writes a schema v1 mailbox message and marks the item in_progress.
  4. When a reply references the work item ID or dispatch message ID, PAH moves it to review.

State file: C:\CODEX PG\CODEX Agent Hub\CODEX state\CODEX_work_items.local.json

Diagnostics and Route Tests

Diagnostics are file-bridge only. They verify readiness without launching live adapters or calling paid services.

Run Diagnostics

  1. Open the Diagnostics tab or use the Communication Test card.
  2. Click Run Diagnostics.
  3. Confirm participants, routes, inboxes, and disabled adapter state pass.

Create a Route Test Ping

  1. Select Codex to Claude or Codex to Claude Code.
  2. Click Create Test Ping.
  3. PAH writes a diagnostic message into the target inbox.
  4. The test passes when PAH later sees a matching reply in CODEX Inbox.

Safety Boundary

PAH's most important job is to coordinate without accidentally authorizing dangerous work.

Graphic snippet: PAH safety is layered. A mailbox message is never enough to authorize protected execution.

Protected Actions

  • Panda Gallery writes
  • Destructive filesystem actions
  • Git commits and pushes
  • Package installs
  • External API calls
  • SMS or email sends
  • Live headless agent runs
  • Paid provider setup
  • Approval-record mutations

Approval Records

Approval records live at:

C:\CODEX PG\CODEX Agent Hub\CODEX approvals\CODEX_approval_records.local.jsonl

They must be exact-path, hash-bound, non-expired, non-revoked, non-consumed, and traceable to an explicit Darrin decision_record addressed to pah.

Headless Adapters

Headless execution remains disabled. PAH has a command contract and MCP config pinning in place for future work, but it does not launch Claude Code headless today.
