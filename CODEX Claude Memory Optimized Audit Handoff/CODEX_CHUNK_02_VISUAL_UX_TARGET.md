# CODEX Chunk 02 - Visual UX Target

## Visual Standard

Audit UI must match Claude's established Panda Gallery testing mockups, not Codex's earlier light dashboard look.

Use:

- Page/shell: `#0d0d18`
- Deep surface: `#14141f`
- Pane surface: `#1a1a2e`
- Hover/surface: `#22223a`
- Border: `#2a2a3e`, `#232336`
- Text: `#e0ddd5`, `#c8c5bb`
- Muted text: `#888`, `#6a6a78`
- Accent/active: `#e8a87c`, `#d4945a`
- PASS: `#5ab87a` or `#7fb069`
- FAIL dot/count: `#d46a6a`
- SKIP: `#8a8a9a`
- Warning: `#f39c12`

Typography:

- `Segoe UI`, `SF Pro`, `Noto Sans`, sans-serif
- 13px body text
- 18px page-level heading
- compact uppercase labels for state/meta text
- 4px to 6px radii
- thin borders

## Required Vocabulary

Use PG testing terms:

- PASS
- FAIL
- SKIP
- Region capture
- Review
- Click to review
- FAIL panel
- Session Package
- Claude Handoff

Avoid introducing a separate SaaS/dashboard vocabulary unless it maps directly to testing workflow.
