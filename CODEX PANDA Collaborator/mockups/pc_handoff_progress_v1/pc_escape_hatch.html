  • Persistent amber banner (state 3) per spec §8.3: "⚠️ Handoff load failed — no handoff summary available. Create or obtain a fresh handoff before serious work begins." Surface = --warn-soft over dark. The banner persists until a fresh handoff lands — it has no dismiss button because the two-step upstream already captured the user's acknowledgment.
  • Statusbar pill "handoff context unknown" uses err-tinted pill (passive chip per Bible §6, not clickable) so the missing-context state is signalled in two places — banner + statusbar — for redundancy without duplication of action.
  • Start Session + Create Safe Handoff stay visibly disabled (grey on dark pane-2 background) per spec §8.3 prerequisites: Start Session activates only when a valid handoff package is loaded; the escape clears handover_pending but does not satisfy that prerequisite.
  • Workflow guide shows all 5 stages in attention state with concise reasons ("complete" / "unknown" / "no context" / "no context" / "awaiting work") — single short label + state per Bible §6.2.
