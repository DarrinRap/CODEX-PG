Bible compliance notes
  • Surfaces: all dark Bible navy — --canvas, --chrome, --pane, --pane-2 (no white/cream — Inviolable Rule #9).
  • Buttons: rectangular, 36px height, radius-md (4px), Bible .gbtn grammar. Done is dismiss-and-reveal only (per §6.2) — neutral secondary, NOT green primary. The safe-action green primary lives on the Outgoing Confirmation screen's "Hand Off to [Name]" button. Retry is danger; View Full Log is neutral.
  • Pills: only on the per-step status column (queued / running / pass / fail). Pills never perform work — they're passive state indicators per Bible §6.
  • Color semantics: green = pass / ready, accent = in-progress, amber = INCOMPLETE warning, red = fail / hard-block. User identity colors NOT used here (this is a procedural surface, not a user-identity surface) — matches PC redesign spec §5.3.
  • Mono fonts used only for precision data — --mono on step numbers, percentages, timestamps, status pills (per Bible §3.4).
  • Hard-block banner uses --err-soft background + --err-border + bold --err text — explicit per spec §7.4.
  • Spinner uses 1.5s linear motion (Bible §4.3 motion lock — no bounce/spring/elastic).
  • Done button disabled in In-Progress and Foundational-Fail states (only available when all steps 1–7b PASS — spec §6.2).
  • Retry scope shown in button text per spec §6.3 / §7.5: "Retry from Step 1" (foundational) vs "Retry Steps 4–5" (component, named subset).
