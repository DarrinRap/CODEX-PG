Bible compliance — spec §8 checklist
  • Surfaces: all dark Bible navy — --canvas #14141f, --chrome #161625, --pane #1a1a2e, --pane-2/--pane-raised #22223a. Zero white/cream/light panel backgrounds.
  • Token discrepancies vs Phase 0: --user2 = #4dd9e0 (Phase 0 has #5fa0a8); --ok = #6da850 (Phase 0 has #7fb069). Per CD ruling 20260504_007100, Phase 2's first commit will update Phase 0 :root to match these spec §5.1 values.
  • Header height: 42px. Workflow rail: 32px. Footer: 28px (40px when emergency). Body: 768 − 42 − 32 − 28 = 666px (or 654px in emergency state).
  • Font ceilings: chrome (header / rail / footer) max 14px (active-user); body max 13px (status-row body-text); section headers 10px mono caps; pills 10–11px mono.
  • Monospace usage per Bible §3.4: branch / SHA / paths / timestamps / counts / package IDs only. UI font for everything else.
  • Buttons (Bible §6 grammar): rectangular, border-radius: var(--radius-md) = 4px, flat surface backgrounds — NO linear-gradient fills. Heights: 28px secondary, 40px Start Session, 48px Create Safe Handoff. The .btn.green class is --ok=#6da850 flat fill, .btn.danger is --err flat fill.
  • Identity color rule (spec §5.3): User 1 amber #e8a87c and User 2 cyan #4dd9e0 used ONLY as identity markers — section header left border stripe (3px), display name text, header active-user dot, workflow rail Step 4 dot, Switch button border. NEVER as button background. The .btn.user1-bordered and .btn.user2-bordered classes confirm this: background: var(--pane-2) always; only the border-color takes the identity color.
  • Switch button states: enabled (other user active) → border 1px solid var(--user1) / var(--user2) vivid; disabled (this user active) → border var(--user1-dim) / var(--user2-dim) = rgba(...,0.25); background never identity-colored.
  • Create Safe Handoff dominance: right-panel 48px height — the largest green action button on the screen by both height and panel area. Visually dominates all other actions per §4.5.
  • Emergency Pause exactly once: header only. State C footer is a passive red BAND showing active state + Clear Pause control — not a duplicate Pause button. Confirmed.
  • Status Messages dominant: ~70% of center column height (flex 1 1 70%); single scrollable container, no nested scroll regions; color-coded 4px left border per row type (info / success / warn / err / wait); largest single surface on the screen.
  • Workflow rail compact: 32px slim progress rail with 5 steps as numbered dots + 2-3 word labels separated by 8px chevrons. No cards, no body copy, no border boxes — confirms §4.2.
  • Zero page-level vertical scroll at 1366×768: .pc-viewport { width:1366px; height:768px; overflow:hidden } hard-locks the viewport. Internal scroll regions are limited to .status-list only (single scroll container per spec §4.4).
  • No 5-equal-column grid: 3-column body (grid-template-columns: 280px 1fr 360px) — Status Messages dominates the flex-grow center; left and right are fixed-width.
  • No truncation at 1366×768: all labels visible; pills don't overflow; user names fit; status messages wrap at row-body width.
