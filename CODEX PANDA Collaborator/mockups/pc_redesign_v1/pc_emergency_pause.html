PC Emergency Pause — operator-controlled safety stop

This is the focused detail view for the Emergency Pause control. In normal operation, PC runs without intervention. The Emergency Pause button in the header (top-right) is always reachable and stops all PC operations on demand.

When to use Emergency Pause

  • You notice PC behaving unexpectedly (UI freeze, unexplained log activity, unusual git operation about to execute).
  • You need to take a phone call mid-handoff and want to guarantee nothing advances while you're away.
  • An automated test is about to write to a path you didn't expect, and you want to inspect before letting it proceed.
  • Any reason at all. PC pauses on demand. No justification required from the UI.

What pause does

  • Halts all background scans, ack polls, and pending writes.
  • Disables every primary action in the UI (Create Safe Handoff, Start Session, etc.).
  • Writes incident evidence to incidents/<ts>.json.
  • Preserves the current session, working tree, and any in-progress handoff package.
  • Never auto-rolls-back, auto-commits, or auto-cleans the working tree. Pause is a freeze, not a revert.

What pause does NOT do

  • Discard work — pause never deletes or rewrites anything.
  • Push to remote — pause is local-only.
  • Force-finish an in-progress handoff — the package stays in its build dir until you resume or discard.
  • Lock files — operator can still edit files manually outside PC; pause is about PC's own activity.

Tip: Press Ctrl + Esc from anywhere in the app to invoke Emergency Pause without reaching for the button.

▶ Resume PC operations?

This will lift the pause and restart all halted streams. The incident JSON will be appended with a resume event, and PC returns to its previous operational state.

PC will:

  • Restart background working-tree scans (next scan in ~30s)
  • Restart setup-test ack polls (if any tester is in setup)
  • Re-arm the daily report rollover timer
  • Re-enable all primary actions (Create Safe Handoff, Start Session, etc.)
  • Append a resume event to incidents/2026-05-05_013142.json

In-progress handoff (pkg_…0115) remains frozen at file-stage 4/8 — you can resume the build manually from the handoff panel after PC resumes.
