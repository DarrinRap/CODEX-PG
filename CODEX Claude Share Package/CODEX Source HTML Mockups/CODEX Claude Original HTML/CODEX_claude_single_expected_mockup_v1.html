Single-expected step layout — mockup v1

Fix for G1/G4 (wasted vertical space) + G2 (button style) + G3 (narrow-resize button truncation).
What this mockup locks in
  • Body gets NO vertical stretch. Uses flex: 0 0 auto + max-height: 40%. Only scrolls if content actually overflows.
  • Spacer absorbs empty space. A flex: 1 1 auto element between body and EXPECTED pushes EXPECTED + button row to the bottom of the pane. Content top, void middle, buttons bottom. Predictable.
  • All bottom-bar buttons are outlined — no solid fills. PASS becomes green-outline (was solid green). FAIL stays amber-outline. Cancel/Pause/Back stay neutral-outline. Entire pane speaks one visual vocabulary.
  • PASS and FAIL get setMinimumWidth(76) and setMinimumWidth(72). Prevents the G3 narrow-resize truncation.
  • Pane MIN_SIZE width bumps from 320 → 380. Five outlined buttons at their floor widths + margins need 380 to fit without overlap.

Code changes this locks in

1. MIN_SIZE = (380, 320) — up from 320 width.

2. button_row: add self._pass_btn.setMinimumWidth(76), self._fail_btn.setMinimumWidth(72).

3. QSS #instructionPanePass: swap solid-fill rule for outlined rule — background-color: transparent; color: #5ab87a; border: 1px solid rgba(90,184,122,0.55);. Hover: background-color: rgba(90,184,122,0.10); border-color: #5ab87a;. Pressed: deeper tint.

4. root.addWidget(self._body_scroll, 1) → root.addWidget(self._body_scroll, 0) (stretch removed) AND self._body_scroll.setMaximumHeight(...) or equivalent SizePolicy tweak so body doesn't expand indefinitely. Add root.addStretch(1) between body-scroll and EXPECTED so extra space absorbs there.
