Stats-row text decision for ACTIVE state (Option A, recommended). During ACTIVE, drop the word “Capturing” from the stats text. The animating bar communicates “capturing” more directly than the word. Non-ACTIVE states retain the full phrase because the dot colour carries less information than the bar does.

Implementation notes baked into this mockup
• Row height pinned at 30 px via min-height on .stats-row. In Qt, use a fixed-height container or set setFixedHeight(30) on a QFrame that holds the HBoxLayout.
• MicLevelBar uses flex: 1 with min-width: 140 px below its Qt-side 200 px minimumSizeHint, because the stats-row layout also has a text column that must coexist. In Qt, set the bar's stretch factor to 1 in the HBoxLayout and leave its minimumSizeHint at 200 px — the layout will honor minimumSizeHint.
• Text column uses flex: 1; min-width: 0; text-overflow: ellipsis so long DISCONNECTED copy truncates gracefully if the user drags the pane to below MIN_SIZE.
• ACTIVE state hides the dot widget (setVisible(False)) and shows the bar; non-ACTIVE hides the bar and shows the dot. The text label updates copy in both branches.
• Non-ACTIVE states share the amber dot intentionally. In a future iteration we might differentiate FAILED (solid red dot?) — defer.
