Unified Action Bar — #107 + #110 bundled fix (Mockup v1)

Target: move all contextual action buttons (PASS, FAIL, SKIP, Next →) out of the scroll area and into the pane-level action bar at the bottom. One row, always reachable, state-dependent content. Fixes #110 (buttons scroll out of view on tall rows) and #107 (two-row footer on checklist steps). Widths shown: 430 px (MIN_SIZE, current floor) and 360 px (below wrap breakpoint) to prove the layout holds across the pane's size range.

Architectural change

Before: In-row _actions_host (PASS/FAIL/SKIP on the active row, inside the scroll area) plus a separate ChecklistStepView._footer (summary + Next →) below the scroll area, plus the pane-level button_row (Cancel / Pause / ← Back) at the bottom. Three horizontal action surfaces, two of them inside the scroll region.

After: One pane-level action bar hosts all buttons. _actions_host is removed from _ChecklistRow entirely. ChecklistStepView._footer dissolves — its Next button moves to the pane bar, and its summary text moves to a new thin summary strip above the action bar (always visible once any item is answered, per recommendation — see Option A vs B below).

Width budget

MIN_SIZE is currently 430 px (pane floor). Adding SKIP means the worst-case button row has 6 buttons: Cancel / Pause / ← Back / PASS / FAIL / SKIP.

6-button budget check (checklist active-row case)

Current 5-button budget: 16 + 70 + 8 + 70 + 8 + 72 + 8 + 76 + 8 + 72 + 16 = 424 px → rounded to 430.

Proposed 6-button budget (adding SKIP at 72 px + 8 px gap): 16 + 70 + 8 + 70 + 8 + 72 + 8 + 76 + 8 + 72 + 8 + 72 + 16 = 512 px.

Recommendation: bump MIN_SIZE width to 520 px (or tighten button minimums). Former is safer; latter is more compact but risks text clipping on PASS/FAIL/SKIP labels.

Decision point for Darrin: (i) bump MIN_SIZE to 520 and keep current button widths; (ii) keep MIN_SIZE at 430 and tighten button widths to ~60–64 px each; (iii) wrap the 6-button row to two rows below a breakpoint.

Pause behavior: all action-bar buttons except Cancel disable when paused (matches today's contract). Keyboard shortcuts P / F / S / Enter / Shift+Enter / Esc remain wired to the same handlers — they now target the pane-level buttons rather than row-internal buttons, but handler routing through ChecklistStepView.handle_pass etc. is unchanged.

First-completion auto-advance: unchanged. When the last pending item is answered and the step transitions to all-answered for the first time, the pane auto-advances (today's behavior). The Next → button is reachable only after a re-answer.

Removed entirely: _ChecklistRow._actions_host widget + all its button setup, ChecklistStepView._footer widget + _next_btn, ChecklistStepView._update_footer method. Summary text still computes inside ChecklistStepView but is rendered by the pane via a new summary_changed signal (payload: {pass, fail, skip} dict).
