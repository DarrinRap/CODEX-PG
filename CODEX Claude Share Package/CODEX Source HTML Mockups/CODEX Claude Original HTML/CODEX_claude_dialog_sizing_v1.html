#129 sizing rule — Settings dialog mockup v1

Visual target for the four-invariant sizing rule. Each scenario shows the Testing — Settings dialog (Pane tab) at a different width. Buttons stay intact across all widths; text wraps at the narrow end; gap between buttons is constant 10px. Default width on first open = floor × 1.15.

The four invariants

  1. Buttons all visible. Every button in the active cluster fully rendered. None clipped, none hidden.
  2. Inter-button spacing fixed. 10px gap between buttons. Never compressed. Window can't shrink past the point where compression would be required.
  3. Text never clipped. Labels word-wrap freely (window grows vertically). No horizontal truncation.
  4. Multi-line inputs ≥ 2 lines. Any QTextEdit has min-height = 2 lines of rendered font + padding.

Implementation notes

compute_min_size() on TestingSettingsDialog returns QSize(min_w, min_h) derived from: button-cluster widths (with fixed gaps), longest single-line label that can't wrap (tab labels, button text), multi-line input floors (lineSpacing × 2 + padding), and vertical padding/spacing constants.

Called from __init__ for initial sizing, and from any layout- affecting state change. setMinimumSize() applied; current geometry honored if user has dragged wider.

Default-size policy on first open (no persisted geometry): resize(min_size × 1.15). Persisted geometry (#128) wins on subsequent opens.

Existing widgets to remove: setMinimumWidth(420) in TestingSettingsDialog.__init__ → derived from compute_min_size now. _FailNoteEdit.setMinimumHeight(60) / setMaximumHeight(84) → replaced with derived floor + uncapped grow (let user resize; floor enforced by invariant 4).
