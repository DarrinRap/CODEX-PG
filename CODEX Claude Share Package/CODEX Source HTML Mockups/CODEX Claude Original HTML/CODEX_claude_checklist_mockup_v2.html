Three pane widths + a live resizable demo. Unified list, no walled cards, width-clamped.
Width constraints — Claude Code must enforce these in code
  • Hard minimum: pane width ≥ 280 px. Apply via setMinimumWidth(280) on the instruction-pane dock widget. Below this, Pass/Fail/Skip collide with the label regardless of wrapping.
  • Wrap breakpoint: at pane width ≤ 360 px, the active row drops its button row below the label (separate line, right-aligned). Implemented via a layout switch keyed off the pane's current width in resizeEvent.
  • Above 360 px: buttons sit inline on the right of the active row.
  • Title bar: step name truncates with ellipsis; step count (2 / 4) never truncates.
