Single-expected step — filler placement v2

Choosing where the flex: 1 spacer goes when body content is short vs. medium. Informed by live-testing screenshots showing the v1 mockup only drew the short-body case.
The conflict that surfaced

v1 mockup (shipped in v3.66): spacer between body and EXPECTED. EXPECTED pinned to bottom of pane, just above button row. Works great for short body text.

Live testing with B1 / B2 (medium-length body): the spacer creates a visible gap between body and EXPECTED. The user reads this as "wasted space" identical to the original G1/G4 bug — we just relocated the void.

Root question: where should the absorbed void go — above EXPECTED, or below EXPECTED?
