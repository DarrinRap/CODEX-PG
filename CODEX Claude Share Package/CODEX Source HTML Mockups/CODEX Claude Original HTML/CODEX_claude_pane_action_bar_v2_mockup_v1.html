§5 Action step (NEW) — pick an Acknowledge variant#125 — the design decision

An action step has a body ("Open File → Preferences.") but no observation to report — the tester just does the thing and advances. PASS/FAIL would be a nonsense claim ("this navigation passed"). Three button treatments below; pick one. All three drop the EXPECTED block entirely since there's nothing to observe — a small amber ACTION marker replaces it for visual consistency.

Design summary & decisions pending

Locked in this mockup:
· Two-row action bar at width ≤ 420 px; single row above. Floor MIN_SIZE = (380, 480).
· Navigation cluster (Cancel / Pause / Back) always left; outcome cluster (PASS / FAIL / SKIP / Next → / Acknowledge) always right.
· New kind: "action" schema discriminator (§4.5 extension to TESTING_SECTION_SPEC.md). Loader XOR now three-way: expected ⊕ checklist ⊕ kind:"action".
· Action steps record outcome: "ACK" in results_latest.json. Not PASS, not SKIP — a distinct fourth outcome.
