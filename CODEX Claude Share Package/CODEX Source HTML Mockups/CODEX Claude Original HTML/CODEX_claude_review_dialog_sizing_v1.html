Part 3 — side-selection algorithm

# Runs in RegionReviewDialog.showEvent, after adjustSize()

pane = parent._instruction_pane
if pane is None or not pane.isVisible():
    center_on(MainWindow)
    return

pane_geo = pane.frameGeometry()
screen_geo = QGuiApplication.screenAt(pane_geo.center()).availableGeometry()
gap = 12

# Which side has more room?
space_left = pane_geo.left() - screen_geo.left()
space_right = screen_geo.right() - pane_geo.right()
need = self.width() + gap

if space_left >= need:
    move to (pane_geo.left() - self.width() - gap, pane_geo.top())
elif space_right >= need:
    move to (pane_geo.right() + gap, pane_geo.top())
else:
    center_on(MainWindow) # neither side fits — fall back
Why vertical-align to the pane's top rather than center? The pane is tall+narrow, the dialog is wider+shorter. Aligning tops (not centers) keeps the dialog from peeking above the screen when the pane is near the top edge. If the pane bottom is close to the screen bottom, we can additionally clamp the dialog so it doesn't extend below screen_geo.bottom() — one extra line in the positioning code.
